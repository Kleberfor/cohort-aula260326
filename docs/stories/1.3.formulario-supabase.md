## Story
**As a** visitante,
**I want** preencher meu nome e WhatsApp e enviar,
**so that** eu receba o contato da assessoria.

## Acceptance Criteria
1. Formulário contém campos `Nome completo` e `WhatsApp (com DDD)` com labels visíveis
2. Validação client-side: ambos os campos são obrigatórios antes do submit
3. Ao submeter, os dados são inseridos na tabela `leads` do Supabase via SDK JS
4. Após envio bem-sucedido, o formulário é substituído pela mensagem: "Recebemos seu contato! Falaremos em breve."
5. Em caso de erro na inserção, exibe mensagem: "Algo deu errado. Tente novamente."
6. Botão exibe estado de loading durante o envio

## Dev Notes
- Supabase client: criar em `lib/supabase.ts`
  ```ts
  import { createClient } from '@supabase/supabase-js'
  export const supabase = createClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
  )
  ```
- Tabela `leads` no Supabase: colunas `id` (uuid, PK auto), `nome` (text), `whatsapp` (text), `created_at` (timestamptz, default now())
- RLS: desabilitado na tabela `leads` para permitir insert anônimo (projeto didático)
- A tabela `leads` deve ser criada manualmente no dashboard do Supabase antes do teste
- Componente `LeadForm` usa `useState` para controlar loading/error/success
- Após sucesso: esconder formulário e mostrar mensagem de confirmação (não redirecionar)
- Usar `"use client"` no componente pois usa hooks React
